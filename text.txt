這個錯誤 Host key verification failed. 
指的是 Jenkins 執行環境不信任遠端伺服器（10.21.105.114）的 SSH 指紋（Host Key）。

在預設情況下，當你第一次透過 SSH 或 SFTP 連線到一台新機器時，
系統會詢問你是否要將這台機器的指紋加入 known_hosts。由於 Jenkins 是自動化執行，
無法進行人工輸入 "yes"，因此連線會直接被拒絕。

以下是三種常見的解決方案，按推薦程度排序：

方案一：關閉主機金鑰檢查（最快、最常用於內網）
在 sftp 指令中加入參數 -o StrictHostKeyChecking=no。這樣 SFTP 就會自動接受該主機金鑰，不會彈出確認提示。

修改程式碼如下：
sh "echo 'put ${JAR_FILE_NAME} /iso' | sftp -o StrictHostKeyChecking=no -i ${SFTP_K} version@${REMOTE_HOST}"

方案二：預先手動掃描並加入信任清單（最安全）
如果你希望保持安全性（避免中間人攻擊），你可以先在 Jenkins Agent 伺服器 上，以 Jenkins 執行的使用者身分（通常是 jenkins）將遠端主機加入 known_hosts。

在 Jenkins 伺服器的終端機執行：
# 切換到 jenkins 使用者 (如果需要)
sudo su - jenkins

# 掃描並加入遠端主機金鑰
ssh-keyscan -H 10.21.105.114 >> ~/.ssh/known_hosts
